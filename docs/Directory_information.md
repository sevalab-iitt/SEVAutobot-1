# Directory Structure — `jetauto_ws`

This document maps out the JetAuto Pro's onboard ROS workspace (`jetauto_ws`) so a new user can find launch files, configs, and source code without needing to explore the robot blindly over SSH.

All output below is captured directly from the robot's terminal — re-run the commands shown if you suspect the structure has changed since this was last updated.

## 2. Top-Level Workspace Layout

Shows the standard catkin workspace folders — `src`, `build`, `devel` (and `install` if present).

**Command:**
```bash
tree -L 1 ~/jetauto_ws
```

**Screenshot:**
> <img width="601" height="204" alt="{00493946-3E08-4C28-9197-55475F3316DE}" src="https://github.com/user-attachments/assets/578f3f28-c16e-4a57-b0cd-4ca0d6dfe3ce" />


**Result:**
```
jetauto_ws/
├── build/      ← compiled build artifacts, not edited directly
├── devel/      ← generated setup files (e.g. setup.bash), not edited directly
└── src/        ← all actual ROS package source code — this is what matters
```

> **Note:** `build/` and `devel/` are auto-generated by `catkin_make`/`catkin build`. New users should never need to manually edit anything inside them.
